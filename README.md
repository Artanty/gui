# gui

## Что это?
input, select etc
для angular mfe (v3: просто mfe)

## Зачем?
Можно изменить их конфигурацию и обновить дизайн элментов приложения без пересборки (v2: в рантайме)

front
 - конструктор ui элементов.
 - модуль, предоставляющий инпуты, селекты и тд.

как работает gui directive body:
1
remote запрашивает у host'а ресурс этого remote'а (gui@web)
`{ "remoteName": "gui", "resourceName": "directive", "busEventId": "bzGkYhIk" }`
2
host возвращает класс GuiDirectiveBody,
3
remote делает инстанс этого класса, передавая в конструктор element, injector, WebComponentWrapperComponent итд
4
remote вызывает у этого инстанса метод await directiveClass._findCustomElement()
5
- gui@web читает tagName и type элемента
- ищет его среди !(пока что)своего мэпа ElementsMap
- вычленяет имя ремоута, которому принадлежит этот элемент
- проеряет, подгружен ли этот ремоут
6
пытается забрать этот элемент, предполагая, что ремоут его задефайнил:
customElements.get(customElementName)

==
Что все что дефайнится - лежит в отдельном mfe модуле (exposed),
потому что внутри основного модуля можно использовать сами компоненты.

host сейчас умеет стучаться в Exposed модуль ремоутов и дефайнить там кастомные элементы.
gui-директива в ремоутах пока что (todo) будет стучаться в главный модуль за компонентом.
